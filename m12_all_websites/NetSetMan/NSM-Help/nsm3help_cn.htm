显示描述

NetSetMan 是一款网络设置管理器，它可以在 6 种不同的网络配置之间轻松切换，可视结构的配置选项包括：

  • IP 地址
  • 网关 (incl. Metric)
  • DNS 服务器
  • WINS 服务器
  • IPv4 & IPv6!
  • 扩展 WiFi 管理
  • 计算机名称
  • 工作组
  • DNS 域
  • 默认打印机
  • 网络驱动器
  • NIC 状态
  • SMTP 服务器
  • Hosts 文件条项
  • 脚本 (BAT, VS, JS, ...)
 附加功能只在 Pro 版中：
  • 只可用于在工作中
  • 配置不限数量
  • 网络-域
  • 完成代理服务器设置
    (Windows/Internet Explorer & Firefox)
  • 浏览器主页
    (Internet Explorer & Firefox)

NetSetMan 提供了一些您在 Windows 中可能错过的功能：
一款简单、明了、易用的管理所有网络设置的强大软件。
试用它，您将会离不开了它了。

所有设置及更改将以特殊文件(settings.ini) 自动保存，故而备份、传输或更新版本绝对没有问题。

只需鼠标点击两下即可启用已保存的配置！

配置

这些标签也就是可用的配置文件。在这些标签中您可以保存 6 组不同的设置。

当启用一个单一的配置时，如果您要在不同网络适配器之间的配置更改，您可以使用多配置 功能。点击下拉列表旁边的“+”按钮，一个新的菜单栏会出现，包括两个“多配置”(当前一个和新建的一、空的一个)。 标题栏会根据网络适配器在“多配置”中选定后自动命名，而且无法更改。您可以创建多个的“多配置”。当配置启用后，所有的配置都一个接一个的被启用。您可以通过拖放方式重新排序。

所有的更改操作都会自动保存，无须担心不慎丢失。

您可以右击要重命名的配置文件并选择 “重命名”。

管理配置更有简单，您可以在菜单栏中使用配置管理。可以通过鼠标或键盘快捷操作编辑“配置”、“多配置”以及“组”(仅限于 NetSetMan Pro )(允许导航键、F2 重命名，等等) 当使用拖放方式排列项目时，您可以会注意一些不可避免的限制，主要给定的技术条件：配置无法删除，多配置无法重命名，配置可以交换但不能移动。复选框表示对应的配置在托盘或精简模式中是否可见。

网络适配器 (NICs)

所有可用的适配器都以其名称和类型显示出来。

为了便于管理 NICs ，您可以通过“刷新”加载所有可用的连接名称到下拉列表中，最后选择一个网络适配器。

如果您要复制设置文件(settings.ini) 到另一台计算机中，NetSetMan 将会自动尝试重新分派配置中的 NICs。但是如果新系统有相同名称的 NIC ，则它只可以运行(截图中图示为“LAN”)。

如果找到的 NIC 名称不机，则它将以“[?]”列出。 这时您必须再刷新查找可用的 NIC 列表。

如果 NIC 不在列表中(可能由于处于禁用状态，或是虚拟，...)，您可以使用高级刷新 功能(点击“刷新”旁边的箭头即可访问)。 这样您就可以找到一个可用的 NIC 完整列表。 检查所有您要的 NICs 是否在列表中。

注意：所有在列表中的 NICs 都在“托盘信息”中显示。

获取当前设置

需在空白区域旁边点击 符号即可载入选定配置的当前设置。
鼠标右击配置 (上面已提及的符号)并选择 “获取所有当前设置”即可获取到所有当前的设置。

清除设置

如果只想清除一个特殊的地址，您只需在它的标题上双击即可。
如果清除当前标签的设置，您首先要选定标签，接着在标签标题上右击并按下“清除项目”。

复制 & 粘贴设置

如果您要在不同的配置中切换，您可以在第一个配置文件点击鼠标右键即可复制、粘贴整个配置文件，选择“复制设置”，然后在第二个配置文件上点击鼠标右键并选择“粘贴设置”即可。

特定的粘贴功能如 IP、DNS 以及 WINs 组都可以使用。点击“开始”图标，复制设置将填入到组中。

 
附加地址

如果在同一时间您需要处于不同的网络类型中，您可以指定多个 IP、网关、DNS 与 WINS 地址。只需在一个 IP 组中点击 “+” 符号。在“附加地址”窗口您可以创建无限个地址。在上方空白区域输入您的值并点击 [ v ] 按钮即可添加地址到列表中。

您可以通过拖放方式进行排序。
高亮显示项目后点击红色的 [ x ] 按钮即可删除操作。
双击项目即可填充地址值到空白区域。
但是如果在主窗口点击 符号，则只读取第一个值，此时它读取所有已分配的地址。

WiFi

NetSetMan 配置可以用于连接至已有的 WiFi 配置。只需在下拉列表中选择一个 WiFi 适配器，然后从 WiFi 下拉列表中选择一个 WiFi 配置。如果您只想断开您的 WiFi 连接，请选择空的项目。

切了基本的 WiFi 切换外，NetSetMan 还提供了一个大批量的 WiFi 管理。点击 WiFi 图标或使用主菜单(工具 > NSM WiFi 管理) 即可访问，您将会找到一个所有当前可用的 WiFi 网络列表。黄色背景的表示存在相同 SSID 的配置。绿色背景的表示当前已经连接的网络。您可以通过双击或“连接”按钮连接一个网络。如果不存在该连接的配置，您可以先行创建。设置 (身份验证、加密...) 是否自动预先设置取决于已识别的网络。 但是，在某些情况下您需要作调整，比如识别到个别原因引起的无法正常运作。

NetSetMan 使用原始的 Windows WiFi 配置文件运作的。 这意味着您可以读取 Windows 中或其他 WiFi 软件中所使用的相同配置。您可以创建、修改或删除 WiFi 配置。

 

要求：

NetSetMan 使用原始的 Winodws WiFi API。这样可以带来一个好处，就是不需要安装新的驱动器(可以导致一些冲突)。另一方面 NetSetMan 的 WiFi 只能运行于支持 WiFi 的 Windows 系统上。值得庆幸的是如今的大部分版本都支持。如果您使用 Vista 或更高版本，那最好不过。如果是 XP，您必须安装 SP2 或 SP3。不支持 Windows 2000

请确认 WLAN AutoConfig (Wlansvc) (Vista 及更高版本) 或r Wireless Zero Configuration (WZCSVC) (XP) 是否在运行。

如果您的 WiFi 适配器使用生产商配套的软件管理 WiFi 设置，则大部分情况下它会阻止 Windows 访问。您必须在软件中找到选项以切换到 Windows 管理。
因为不同的生厂商都有自己的软件，不是通用。 比如：如果您使用 AVM FRITZ!WLAN USB 设备，点击软件的托盘图标选择“Windows WLAN”。

脚本

NetSetMan 为您提供了一个管理脚本文件的便捷方式。但它没有自身的脚本语言。也就是说，您只能使用您操作系统中所能运行的脚本语言。常规的脚本语言多为VBS、JS 以及简单的 DOS 批处理文件。使用您喜欢的搜索引擎即可查找脚本以及使用该脚本您可以做什么的更多信息。

打开已有的脚本文件或创建新脚本文件即可进行编辑。
在所有其他的设置更改前后，您可以执行两种脚本。
如果您需要在运行脚本和更改网络设置之间延迟，您可以在底部进行设定。

映射网络驱动器

使用 NetSetMan 您可以映射或断开无数个网络驱动器。
请在窗口底部点击 “[显示帮助]”阅读操作说明。

当启用配置时，选定的 NetSetMan 功能将比较所有当前映射驱动器列表与已经映射的驱动器列表。所有已经映射的驱动器都在两个列表中 (相同的盘符和相同的路径) 将会原封不动地保留。在新映射前，所有其他的网络驱动器将不会被映射。

启用配置

您必须点击 “启用”才可以应用当前配置的所有设置。
注意：只有选中复选框才可启用设置。

将会弹出新窗口。窗口包含当前启用状态的信息。
您可以查看哪些设置被应用以及它们是否成功生效。
您可以点击 “取消” 按钮取消启用操作。 这样当前点下面的设置不会生效。

请注意：即使状态结果正常，最好要重启一下计算机或等待几分钟以使网络同步。(特别 “计算机名” 与 “工作组/域” )在一些特殊情况时，您最好重启您的系统。

精简模式 & 完整模式

除了“完整模式”界面可以更改配置的所有文件外，附加的界面“精简模式”也可用。在右侧的主菜单中点击相应的按钮，您可以随时在两个模式之间切换。

当所有配置文件创建以后，精简模式是最理想的界面。这里您可以精简查看配置文件的所有设置而无需担心不小心的修改设置。

在某些情况下您可以使用 NSM 管理 (“更改配置设置”未选中)精简模式将被自动使用，以防止用户修改设置。

托盘中的 NetSetMan & 托盘信息

NetSetMan 最小化后将出现系统托盘栏中(时针的旁边)。
您可以快速启用一个配置。 只需要右击图标并选择要启用的配置即可。

如果您要将某些配置不显示托盘中，您可以单独将其隐藏起来。在主窗口选择配置后，在标签上点鼠标右键并点击“在托盘以精简模式启用”。 文字旁边的小符号显示该配置将在托盘中启用。该选项在精简模式中显示也会使配置生效。

如果您将光标悬浮在托盘中 NetSetMan 图标上，托盘信息 将会出现。它包含了系统所使用的当前 IP 信息。由于数据是实时收集的，您可以会感受到一些延迟。为了减少延迟时间，您可以考虑将“参数设置的”其他附加信息，如公共 IP、MAC 地址等关掉。

   
备注

如果您需要记住或说明某个配置的目的，您可以为每个配置创建简短的备注。如果配置的备注创建后，它的图标在菜单栏中将以彩色显示。备注只有在完整模式可见，以图标作为提示，而在精简模式中，在配置设置的最上方。

     
自动切换

只需在托盘菜单点击即可轻松启用配置。但在某些场合，特定的位置 总是需要启用特定的配置。虽然通过点击可以完成，但也需要多次的操作。

NetSetMan 自动切换就是解决这个问题的！ 自动切换通过分析 WiFi 附近的网络可以识别您的位置。 您所需要的操作就是用 NetSetMan 的任意一个配置连接一个或多个 WiFi 网络。区域内的网络通过“自动切换”功能将自动启用您的配置。提醒是以预先设置的延时倒计时方式显示，您可以中断启用操作。

工作原理：

首先您必须启用自动切换。 该设置是有效的程序范围时意味着您不必单独启用每个配置。一旦启用了，您可以配置要自动启用的每个配置文件。

为了使用配置进行连接 WiFi 网络，网络必须在配置时处于区域内。 这意味着您只可以在同一位置创建一个配置的“自动切换”，该配置在以后也可以使用。

在检查完 WiFi 网络后决定是否需要区域内的所有网络或其中一个就足够了。 也决定是否网络能被它们的名称(在不同的环境中名称可能相同)或 唯一的 MAC 地址(BSSID)所识别。延迟提示您所要花费的时间，一旦“自动切换”识别到一个适合的位置，您必须要放弃启用自动的配置。

必须注意：

  • 如果 AutoSwitch 规则中有多个的配置，只有第一个才会生效。
  • 完整功能只提供给 Vista 及更新版本 (Win7, ...) 这里 NetSetMan 每隔一段时间就会接收系统执行 WiFi 扫描的提示。自动切换不会执行自身的扫描，这意味着它不会消耗系统的额外资源。
  • 在 XP 无法接收 WiFi 网络中的提示，故而“自动切换”只检查特定的特别事件(程序启动、WiFi 连接状态、WiFi 初始化)。
  • 为了节省系统资源，NetSetMan 不会对您当前设置与即将启用的配置设置进行比较。这意味着如果你要手动更改您的设置，NetSetMan “自动切换”不会注意到变更操作。
  
启用命令行

在一些特殊场合您可能不会使用 NetSetMan 的GUI 图形化界面来更改您的配置。(如果您想把它加载到启动光盘、添加到自动运行菜单，快捷更改配置等)

-h / -hide：最小化运行 NetSetMan

-c / -compact：精简模式运行 NetSetMan
-m / -multi：运行多个 NetSetMan 实例。 注意： 所有的实例写入到同一个设置文件中！

管理模式 & NSM 服务

如果您是以限制权限的用户身份运行 NetSetMan，它可能无法正常运行。但如果您想通过输入帐户密码方式以管理员权限运行程序(这些帐号信息将被加密保存在设置文件中)，NetSetMan 将根据帐号信息依照相应权限启动自身。

第二次方式就是使用 NSM 服务 ，访问网站 www.netsetman.com 免费使用。

它是一个提供 NetSetMan 所需权限的指定服务。如果您要安装服务，您必须有管理员权限。 卸载和普通软件一样，可以在 Windows “添加/删除程序”中操作。 您可以使用服务对话框 (services.msc) 控制服务 - 停止或重新启动。
如果发生错误提示“无效签名”，请确认程序可执行文件是否被修改，组策略是否“关闭自动启动证书升级”被禁用用。

请认真阅读 NetSetMan FAQ 中受限用户的相关描述。

NSM 管理

如果您要将 NetSetMan 设置为其他人也可以使用，您需要指定一下哪些是其他人可以允许的操作。打开 NSM 管理 (参数设置 > 管理) 即可进行操作。 (请不要与“管理模式”混淆)

当您启用该模式时，您必须输入密码，长度为 6 个或更多字符。只有使用密码，您才能更改 NSM 管理设置或执行一些安全的操作 (运行 NetSetMan、启用配置、更改配置以及更新程序设置)。

您的密码校验和将被写到注册表中。请确认普通用户没有权限写入注册表 HKLM\Software\NetSetMan。如果您忘记了密码，您需要手动删除该键值。

为了安全起见，您可以加密设置文件。如果未加密，该文件可以手动编辑。
不过，请记住：不可能是 100% 安全。 故而，即例加密了，那也只是某个程度上的安全。

NetSetMan Pro 附加选项
配置组

有了 NetSetMan Pro，您可以创建无数个配置组。每个组包括 6 个配置。

如果要添加、删除、重新排列或重命名配置组，请使用菜单栏中的 配置管理 。

浏览器 / 代理服务器

有了 NetSetMan Pro，您可以管理 Internet Explorer 与 Firefox 的代理服务器设置。
因为 Firefox 支持不同的配置，每个 Firefox 项目以括号显示配置的名称。
重置应该不解自明，因为它与您的浏览器非常相似。

如果您没输入浏览器设置，您将在主窗口看到 “(...)”，否则选定的浏览器(IE = Internet Explorer, FF = Firefox) 及 HTTP-代理服务器将在 “浏览器” 和 “C”按钮之间变得可见。

如果您在 IE 或 FF 中指定“不使用...代服务器”的地址，您需要输入这样格式 (如“10.*.*.*，10.0.0.0/8”)。 NetSetMan 将会自动过滤浏览器的无效值。

请注意 Firefox 必须 要关闭后方可使新设置生效。

如果您的 Firefox 配置文件夹变更 (比如：新装系统等)，您需要重新选择列表中的配置以便于程序及时更新您的设置文件。

域

如果要加入域，需要输入域的名称、您的用户名及密码。如果要输入密码，只需在“P”按钮上点击即可。现在您可以输入您的密码，该密码将被加密保存在 ini 文件中。不管是您还是其他任何人都无法直接读取该文件。

通常只需输入纯用户名就可以了。
如果发生错误，您可以试试下面两个标准之一作为您的用户名：
1. NETBIOS: DOMAIN\user
2. UPN: user@domain.com
如果您不确定，可以咨询您的管理员。
